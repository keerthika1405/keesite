Sorting

Sorting refers to the process of arranging a list or a sequence of given elements(that can be numbers or strings, etc.) or data into any particular order, which may be ascending or descending.
sorting in data structure:
  • Bubble sort
  • Insertion Sort
  • Selection Sort
  • Merge Sort
  • Quick Sort
  • Counting bSort
  • Heap Sort
  • Radix Sort
  • Bucket Sort

eg:
Suppose, you are trying to find the phone number of your friend in a call-book diary of 100 pages, containing around 500 contact number details. Now, if all of the contacts are arranged in a haphazard manner, without any particular order, then it might take you the whole day to find out the contact detail of 1 person. So, this is where sorting comes into the picture. If the contact names would be arranged in a manner where their names were alphabetically sorted, then it might have taken less than a min to find out the contact number.



Why we need sorting?

  • Reduce the computational complexity of various problems.
  • Make it easier to implement search algorithms to find or retrieve an item from a dataset efficiently
  • Make raw data simpler and meaningful to analyze


Time and Space complexity

Time complexity in the programming world measures the approximate time taken by a piece of code, or algorithm to execute(or run) based on the size of input passed to it. Time complexity does not give us the exact measure of time, but it gives us an overall estimation of the time taken.
The time taken mainly depends upon the number of times lines of code will iterate.

calculating the time complexity

  • Big oh Notation, O:

    • It measures the upper limit of an algorithm's running time or the worst-case time complexity

    1. O(1) means constant time complexity. It means, that no matter what the size of the input is, the time taken to compute it will always be constant
    2. O(N) means linear time complexity, for an input of size N
    3. O(logN) is logarithmic time complexity.
    4. O(N2),O(N3) If our code has a big-Oh time complexity of , etc. then that means it has quadratic time complexity.
  • Omega Notation, Î©:

    • It measures the minimum time taken by algorithms to execute, and calculates the best case time complexity

  • Theta Notation, Î¸:

    • It measures the average time taken by an algorithm to execute. Or, the lower bound and upper bound of an algorithm's running time.

Space Complexity:

Space Complexity is the total memory space required by any program for the execution of its code. It depends on the size of the data structure we are using to do our code. The sorting algorithms which take constant space are also known as the in-place sorting algorithms. If we do not use any data structure(eg. array, list etc.) then our code uses constant space. O(1) specifies constant space complexity. Other parameters like linear, quadratic, or exponential space complexity are similar like what we defined for our time complexity.

In-place Sorting and Out-of-place Sorting

An In-place algorithm modifies the inputs, which can be a list or an array, without using any additional memory. As the algorithm runs, the input is usually overwritten by the output, so no additional space is required eg:insertion sort, selection sort, quick sort, bubble sort, heap sort, etc. they take constant space, the space complexity of these algorithms is O(1).

An algorithm that is not in place is called a not-in-place or out-of-place algorithm. The Out-of-place sorting algorithm uses extra space for sorting, which depends upon the size of the input. merge sort algorithm is an example of the out-of-place algorithm as it requires O(n) extra space for merging.

Stable Sorting and Non-Stable Sorting

The Stable sorting algorithms preserve the relative order of equal elements. sorting algorithms are stable by nature, such as Merge Sort, Timsort, Counting Sort, Insertion Sort, and Bubble Sort.

In a Non-Stable sorting algorithm the ordering of the same values is not necessarily preserved/maintained after sorting. Selection sort, Shell sort, Quicksort, and Heapsort may be considered an example of Non-Stable sorting algorithms.

Adaptive Sorting and Non-Adaptive Sorting

If the order of the elements to be sorted of an input array matters (or) affects the time complexity of a sorting algorithm, then that algorithm is called the â€œAdaptiveâ€ sorting algorithm. Examples of Adaptive Sorting algorithms may include Bubble Sort, Insertion Sort and Quick Sort.

In Non-Adaptive Sorting, the order of the elements in the input array does not matter, the time complexity remains always the same for any order of the given input. n-adaptive sorting algorithms are: Selection Sort, Merge Sort, and Heap Sort.

Internal and External sorting

If the data sorting process takes place entirely within the RAM of a computer, it is called internal sorting. We can only perform internal sorting when our size of data is small enough to be fitted in our main memory. Some algorithms which use internal sorting may include bubble sort, insertion sort, quick sort, etc.

Eternal sorting is used, whenever we need to sort very large datasets, which may not be stored in the memory as a whole at once.

Other Characteristics of Sorting algorithms

Comparison Sort: Comparison sorting algorithms are those which work by comparing each pair of adjacent elements of an iterable eg: Bubble sort, Insertion sort, Merge sort, heap sort etc. are comparison-based sorting algorithms. Whereas, count sort, radix sort, and bucket sort are not comparison-based.

Parallelism: Sorting algorithms are either serial or parallel. A parallel algorithm can do multiple operations at a time, whereas the serial sorting algorithms work on one operation at a time. The serial sorting algorithms work like, one operation is executed only after the previous operation is completed Bucket sort is a parallel sorting algorithm because we can sort the buckets parallelly.

Recursive: Some algorithms are recursive by nature, for example, the merge sort or quick sort, use the divide and conquer technique to solve the problem.

Trade-Offs of Sorting Algorithms

The space and time complexity of an algorithm are the major determinants of its efficiency. To make the algorithm more efficient, we consistently try to reduce both the time and space complexity. But unfortunately, this is not always possible because of the underlying trade-off between the time and space complexity of any sorting algorithm. By that, we mean that, whenever we are trying to reduce one of them (time or space), the other factor(such as time or space) usually increases. Suppose, if we are trying to optimize the space, then the time complexity might increase and vice versa. Hence, we may say there is a trade-off between the sorting algorithms where we can prioritize one of the time and space complexity and implement our algorithm.

                  String pivot = arr[high];
                  int i = (low - 1);

                          // Swap arr[i] and arr[j]

                  String temp = arr[i + 1];
                  arr[i + 1] = arr[high];
                  arr[high] = temp;
          
                  return i + 1;

By comparing with middle element of a sorted array
this technique locates desired elements. When a match is detected, the element's index is returned.

process:
When the middle element of the array is greater than the element that is to be searched, then it seeks for the middle element of the left sub-array. If the middle element is less than the element that is to be searched, then it seeks for it in the right sub-array. It keeps looking for an item until it finds it or the size of the sub-array approaches zero

Space Complexity is the total memory space required by any program for the execution of its code. It depends on the size of the data structure we are using to do our code. The sorting algorithms which take constant space are also known as the in-place sorting algorithms. If we do not use any data structure(eg. array, list etc.) then our code uses constant space. O(1) specifies constant space complexity. Other parameters like linear, quadratic, or exponential space complexity are similar like what we defined for our time complexity.

In-place Sorting and Out-of-place Sorting

An In-place algorithm modifies the inputs, which can be a list or an array, without using any additional memory. As the algorithm runs, the input is usually overwritten by the output, so no additional space is required eg:insertion sort, selection sort, quick sort, bubble sort, heap sort, etc. they take constant space, the space complexity of these algorithms is O(1).

An algorithm that is not in place is called a not-in-place or out-of-place algorithm. The Out-of-place sorting algorithm uses extra space for sorting, which depends upon the size of the input. merge sort algorithm is an example of the out-of-place algorithm as it requires O(n) extra space for merging.

Stable Sorting and Non-Stable Sorting

The Stable sorting algorithms preserve the relative order of equal elements. sorting algorithms are stable by nature, such as Merge Sort, Timsort, Counting Sort, Insertion Sort, and Bubble Sort.

In a Non-Stable sorting algorithm the ordering of the same values is not necessarily preserved/maintained after sorting. Selection sort, Shell sort, Quicksort, and Heapsort may be considered an example of Non-Stable sorting algorithms.

Adaptive Sorting and Non-Adaptive Sorting

If the order of the elements to be sorted of an input array matters (or) affects the time complexity of a sorting algorithm, then that algorithm is called the â€œAdaptiveâ€ sorting algorithm. Examples of Adaptive Sorting algorithms may include Bubble Sort, Insertion Sort and Quick Sort.

In Non-Adaptive Sorting, the order of the elements in the input array does not matter, the time complexity remains always the same for any order of the given input. n-adaptive sorting algorithms are: Selection Sort, Merge Sort, and Heap Sort.

Internal and External sorting

If the data sorting process takes place entirely within the RAM of a computer, it is called internal sorting. We can only perform internal sorting when our size of data is small enough to be fitted in our main memory. Some algorithms which use internal sorting may include bubble sort, insertion sort, quick sort, etc.

Eternal sorting is used, whenever we need to sort very large datasets, which may not be stored in the memory as a whole at once.

Other Characteristics of Sorting algorithms

Comparison Sort: Comparison sorting algorithms are those which work by comparing each pair of adjacent elements of an iterable eg: Bubble sort, Insertion sort, Merge sort, heap sort etc. are comparison-based sorting algorithms. Whereas, count sort, radix sort, and bucket sort are not comparison-based.

Parallelism: Sorting algorithms are either serial or parallel. A parallel algorithm can do multiple operations at a time, whereas the serial sorting algorithms work on one operation at a time. The serial sorting algorithms work like, one operation is executed only after the previous operation is completed Bucket sort is a parallel sorting algorithm because we can sort the buckets parallelly.

Recursive: Some algorithms are recursive by nature, for example, the merge sort or quick sort, use the divide and conquer technique to solve the problem.

Trade-Offs of Sorting Algorithms

The space and time complexity of an algorithm are the major determinants of its efficiency. To make the algorithm more efficient, we consistently try to reduce both the time and space complexity. But unfortunately, this is not always possible because of the underlying trade-off between the time and space complexity of any sorting algorithm. By that, we mean that, whenever we are trying to reduce one of them (time or space), the other factor(such as time or space) usually increases. Suppose, if we are trying to optimize the space, then the time complexity might increase and vice versa. Hence, we may say there is a trade-off between the sorting algorithms where we can prioritize one of the time and space complexity and implement our algorithm.

By comparing with middle element of a sorted array
this technique locates desired elements. When a match is detected, the element's index is returned.

process:
When the middle element of the array is greater than the element that is to be searched, then it seeks for the middle element of the left sub-array. If the middle element is less than the element that is to be searched, then it seeks for it in the right sub-array. It keeps looking for an item until it finds it or the size of the sub-array approaches zero
